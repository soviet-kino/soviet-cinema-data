# CLAUDE.md

> Инструкция для AI-ассистентов (Claude Code и др.), работающих с этим проектом.
> Человеческое README — отдельно, в README.md.

## О проекте

**Название (рабочее):** Soviet Bloc Cinema — открытый исследовательский портал о кино СССР и стран социалистического лагеря XX века.

**Миссия:** собрать структурированную базу фильмов, режиссёров, символов, мотивов и референсов, а также корпус исследовательских разборов, раскрывающих второй смысловой ряд, эзопов язык и культурный контекст этого пласта кинематографа.

**Аудитория:** киноведы, синефилы, исследователи культуры, студенты киновузов, переводчики, авторы видеоэссе.

**Принципы:**
- Полностью open source, бесплатный хостинг.
- Данные отделены от приложения, версионируются в git.
- Качество разметки важнее охвата. 200 глубоко размеченных фильмов > 50 000 заглушек.
- Атрибуция и корректные лицензии для всех внешних данных.
- Русский — основной язык интерфейса и разборов; оригинальные названия фильмов сохраняются на языке производства.

## Гео- и временной охват

Страны (на момент производства фильма):
- СССР (включая все союзные республики)
- ПНР — Польша
- ЧССР — Чехословакия (позже Чехия и Словакия до 1992)
- ГДР — Восточная Германия
- СФРЮ — Югославия
- НРБ — Болгария
- ВНР — Венгрия
- СРР — Румыния
- НРА — Албания
- МНР — Монголия (опционально, второй приоритет)

Период: 1917–1991 как ядро. Допускается до 2000 для фильмов, тематически и стилистически продолжающих традицию (постсоветское осмысление, последние работы режиссёров эпохи).

## Структура репозиториев

Проект разделён на несколько публичных репозиториев:

- `soviet-cinema-data` — структурированные данные (фильмы, люди, студии, символы). JSON/YAML. **Лицензия: CC BY-SA 4.0.**
- `soviet-cinema-essays` — авторские разборы и эссе. MDX. **Лицензия: CC BY-NC-SA 4.0.**
- `soviet-cinema-web` — веб-приложение (Next.js). **Лицензия: MIT.**
- `soviet-cinema-tools` — скрипты импорта, валидации, обогащения. Python. **Лицензия: MIT.**

Этот CLAUDE.md живёт в корне каждого репозитория с локальными уточнениями.

## Стек технологий

**Когда нужно писать код, использовать следующее (если пользователь не указал иное):**

- **Frontend:** Next.js 15+ (App Router), React 19, TypeScript, Tailwind CSS.
- **БД (рантайм):** SQLite, генерируется при сборке из файлов `soviet-cinema-data`. Для запросов — Drizzle ORM или прямой better-sqlite3.
- **Поиск:** на старте — встроенный FTS5 в SQLite. При росте — Meilisearch.
- **Контент:** MDX для эссе, парсинг через `next-mdx-remote` или contentlayer-аналог.
- **Скрипты импорта/обогащения:** Python 3.11+, `requests`, `pydantic` для валидации схемы, `SPARQLWrapper` для Wikidata.
- **Внешние API:** Wikidata (SPARQL), Wikipedia (MediaWiki API), TMDB.
- **Хостинг:** Cloudflare Pages или Vercel (фронт). Данные — внутри билда.
- **CI:** GitHub Actions для валидации схемы данных, линта, тестов, деплоя.

**Чего избегать:**
- Тяжёлых решений «на вырост» (Kubernetes, микросервисы, Elasticsearch) до того, как они действительно нужны.
- Платных сервисов без бесплатного тарифа, покрывающего разумные нужды.
- Парсинга источников, чьи ToS это запрещают (Кинопоиск, Letterboxd как основа базы).

## Схема данных

Каждая сущность — отдельный файл. ID = slug на латинице, стабильный, не меняется после публикации.

### Фильм (`films/<slug>.yaml`)

```yaml
id: zerkalo-1974
title_ru: Зеркало
title_original: Зеркало
title_translit: Zerkalo
title_en: The Mirror
year: 1974
country: [SU]
republic: RSFSR              # для фильмов СССР — союзная республика производства
studio: [mosfilm]
director: [tarkovsky-andrei]
screenwriter: [tarkovsky-andrei, misharin-aleksandr]
cinematographer: [rerberg-georgy]
composer: [artemyev-eduard]
cast:
  - person: terekhova-margarita
    role: Мать / Наталья
  - person: yankovsky-oleg
    role: Отец
runtime_min: 107
language: [ru]
genre: [drama, autobiographical]
color: color_and_bw
release_date: 1975-03-07
production_status: released
censorship_status: limited_release    # released | shelved | banned | limited_release | reedited
external_ids:
  wikidata: Q474662
  imdb: tt0072443
  tmdb: 8917
sources:
  - https://www.wikidata.org/wiki/Q474662
schema_version: 1
```

### Человек (`people/<slug>.yaml`)

```yaml
id: tarkovsky-andrei
name_ru: Андрей Тарковский
name_original: Андрей Арсеньевич Тарковский
name_translit: Andrei Tarkovsky
birth: 1932-04-04
death: 1986-12-29
nationality: [SU]
roles: [director, screenwriter]
external_ids:
  wikidata: Q42735
schema_version: 1
```

### Студия (`studios/<slug>.yaml`)

```yaml
id: mosfilm
name_ru: Мосфильм
name_original: Мосфильм
country: SU
founded: 1923
external_ids:
  wikidata: Q1339661
schema_version: 1
```

### Символ / мотив (`motifs/<slug>.yaml`)

```yaml
id: rain-as-memory
name_ru: Дождь как память
description_ru: >
  Дождь у Тарковского регулярно сопровождает сцены воспоминаний и снов,
  становясь визуальным маркером перехода из настоящего в прошлое.
category: [visual, temporal]
schema_version: 1
```

### Референс (`references/<slug>.yaml`)

Связь «фильм → фильм» или «фильм → произведение».

```yaml
id: zerkalo-to-wild-strawberries
source_film: zerkalo-1974
target:
  type: film
  ref: wild-strawberries-1957
kind: thematic              # thematic | visual_quote | structural | musical | dialogue
description_ru: >
  Структура воспоминаний и сновидений отсылает к «Земляничной поляне» Бергмана.
confidence: high            # high | medium | speculative
sources:
  - essay: zerkalo-memory-layers
schema_version: 1
```

### Эссе / разбор (`essays/<slug>.mdx`)

Frontmatter + MDX-контент. Лежит в `soviet-cinema-essays`.

```yaml
---
id: zerkalo-memory-layers
title: Слои памяти в «Зеркале»
author: redaktor-slug
films: [zerkalo-1974]
motifs: [rain-as-memory, mirror-as-threshold]
references: [zerkalo-to-wild-strawberries]
published: 2026-05-20
reading_time_min: 12
license: CC-BY-NC-SA-4.0
schema_version: 1
---
```

## Контролируемые словари

Закрытые списки значений лежат в `vocabularies/`:
- `countries.yaml` — ISO-подобные коды для соцстран (SU, PL, CS, DD, YU, BG, HU, RO, AL, MN).
- `republics.yaml` — союзные республики СССР.
- `genres.yaml`
- `roles.yaml` — роли в производстве.
- `motif_categories.yaml`
- `reference_kinds.yaml`
- `censorship_statuses.yaml`

Любое новое значение — через PR в словарь, не на лету.

## Конвенции

**Slug-и:** kebab-case, латиница, для людей — `familiya-imya`, для фильмов — `nazvanie-god`. Транслит по системе BGN/PCGN для русского. Стабильность slug-а важнее красоты.

**Даты:** ISO 8601 (`YYYY-MM-DD`). Если известен только год — `1974`. Год + месяц — `1974-03`.

**Языки полей:** многоязычные поля имеют суффикс языка (`title_ru`, `title_original`). `_original` — на языке производства в его оригинальной графике.

**Источники:** любое нетривиальное утверждение в YAML или эссе сопровождается полем `sources` со ссылками. Wikipedia — допустимый источник для базовых фактов, но не для интерпретаций.

**Атрибуция внешних данных:** для фактов, перенесённых из Wikidata и Wikipedia, в README соответствующего раздела указывается источник и лицензия.

## Работа с изображениями

- Постеры и кадры **не хранятся** в репозиториях.
- Ссылки на постеры идут через TMDB CDN с обязательной атрибуцией TMDB в футере сайта.
- Скриншоты для разборов используются по принципу добросовестного использования (fair use / цитирование), с указанием фильма, тайминга и правообладателя.
- В YAML фильма допустимо поле `poster_tmdb_path` со ссылкой на путь в TMDB.

## Типовые задачи и как их выполнять

### «Добавь фильм X»

1. Сначала проверь, нет ли уже файла `films/<возможный-slug>.yaml`.
2. Запроси данные из Wikidata по названию и году (если есть Q-ID — сразу по нему).
3. Заполни YAML по схеме выше. Незаполненные поля **пропускай**, не ставь пустые строки.
4. Проверь, существуют ли упомянутые люди и студии. Если нет — создай заглушки в `people/` и `studios/` с минимальным набором полей.
5. Прогони валидатор схемы перед коммитом.

### «Импортируй фильмы СССР за 1970-е»

Это задача для `soviet-cinema-tools`. Скрипт на Python:
1. SPARQL-запрос к Wikidata: страна = СССР, год выпуска в диапазоне 1970–1979.
2. Для каждой записи — генерация YAML по схеме.
3. Запись в `soviet-cinema-data/films/`.
4. Сбор уникальных режиссёров и студий, генерация заглушек в `people/` и `studios/`.
5. Отдельный отчёт о пропусках и конфликтах в `reports/`.

Не пушить десятки тысяч файлов одним PR. Разбивать по годам или республикам.

### «Напиши разбор фильма»

Это задача редактора-человека, не AI. AI допускается использовать только для:
- Поиска возможных референсов (с пометкой `confidence: speculative` до проверки человеком).
- Структурной разметки уже написанного редактором текста.
- Подбора цитат из открытых источников.

AI **не пишет** интерпретации и культурологические выводы самостоятельно. Это принципиально для доверия к проекту.

### «Найди референсы между фильмом X и другими»

1. Прочитай существующие эссе про X.
2. Прочитай эссе про режиссёра X (его влияния и круг).
3. Предложи список возможных референсов с `confidence: speculative`.
4. Все предложения уходят на ревью человеку перед мерджем.

## Валидация

В `soviet-cinema-tools/validators/`:
- Pydantic-модели для каждой схемы.
- Проверка целостности ссылок (упомянутый `director` существует в `people/`).
- Проверка словарных значений.
- Проверка уникальности ID.

CI запускает валидатор на каждый PR в `soviet-cinema-data`.

## Чего AI делать не должен

- Не выдумывать факты о фильмах, людях, датах. Если данных нет — оставить поле пустым и пометить TODO.
- Не сочинять интерпретации второго смыслового ряда от своего имени.
- Не парсить источники, ToS которых это запрещают.
- Не коммитить файлы, не прошедшие валидатор.
- Не менять схему данных без обсуждения — `schema_version` бампится через отдельный PR с changelog.
- Не загружать в репозитории защищённые авторским правом изображения, тексты сценариев, субтитры.
- Не использовать данные из Wikipedia без указания источника в `sources`.
