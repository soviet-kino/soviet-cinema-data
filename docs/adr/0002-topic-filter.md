# ADR-0002: Декларативный фильтр для тем (Topic.filter)

- **Статус:** принято
- **Дата:** 2026-05-18
- **Версия схемы:** 1 (additive, без bump)

## Контекст

Тематические разделы (`topics/`) изначально привязывались к фильмам только
явно — через поле `Film.topics: [<slug>, ...]`. Это работает для редких
тем с кураторской подборкой («хрононавтика»), но не масштабируется на
«естественные» темы:

- «Кино Тарковского» — все фильмы, где Тарковский режиссёр или сценарист;
- «Брежневское кино» — все советские фильмы 1964–1985;
- «Кино Стругацких» — все экранизации Стругацких (через `references`).

Размечать вручную тысячи фильмов в такие темы непрактично. Атрибут
очевиден: достаточно одного запроса по существующим полям.

## Решение

Сущность `Topic` получает опциональное поле `filter: TopicFilter`:

```yaml
id: films-by-tarkovsky
name_ru: Кино Тарковского
description_ru: …
filter:
  director: tarkovsky-andrei
```

```yaml
id: stagnation-cinema
name_ru: Застой
description_ru: …
filter:
  year_from: 1964
  year_to: 1985
  country: SU
```

Поля `TopicFilter`:

| Поле            | Семантика                                     |
|-----------------|-----------------------------------------------|
| `year_from`     | Включительно. Может быть один, без `year_to`. |
| `year_to`       | Включительно.                                  |
| `director`      | Slug персоны — есть в `Film.director`.        |
| `screenwriter`  | Slug персоны — есть в `Film.screenwriter`.    |
| `book_author`   | Slug персоны — автор книги в `References` с `kind: adaptation` и `target.type: book`. |
| `country`       | Код страны — есть в `Film.country`.           |

Все поля опциональны. Заданные комбинируются как `AND`.

Подборка фильмов темы в UI — **объединение**:

1. фильмы с `topics: [topic_id]` (явная привязка);
2. фильмы, удовлетворяющие `filter`.

Это даёт лучшее из двух миров: ручная курация остаётся возможной (можно
явно добавить редкое исключение в любой топик), а массовая разметка не
требуется.

## Альтернативы

1. **Только ручная разметка `Film.topics`.** Не масштабируется на
   1000+ фильмах; редактор не успеет.
2. **Отдельная сущность `Series` / `Collection`** с пред-вычисленным
   списком фильмов. Усложняет схему, требует синхронизации при добавлении
   нового фильма в базу.
3. **Граф запросов / Cypher / SPARQL** — слишком тяжело для проекта на
   YAML + SQLite. SQLite JSON1 справляется на текущих ~3.5k фильмах.

## Последствия

- `Topic.filter` — опциональное поле. Старые топики без `filter` работают
  как раньше.
- Подборка вычисляется на лету через JSON1 (`json_each(films.data.director)`
  и т.п.). На 1k–10k фильмах быстро. При росте каталога до 50k+ нужно
  будет завести материализованную таблицу `topic_film_match`.
- Schema-version не бампим: добавление поля совместимо вниз.
- Редактор может комбинировать декларативный фильтр и явные `Film.topics` —
  например, к динамической теме «Кино Тарковского» добавить
  «Жертвоприношение» (если оно в базе с другим QID на режиссёра).
